# SLM Lexical Substrate

This directory builds the local lexical substrate for SLM.

The database is a build-time digestion forge. The first-class runtime artifact is
the generated SLM Lexical Substrate Pack:

```powershell
python scripts\download_open_english_wordnet.py
python scripts\build_lexicon_db.py
python scripts\build_lexical_substrate_pack.py
```

Generated output:

- `data/slm_lexicon.sqlite`
- `data/slm_lexical_substrate/manifest.json`
- `data/slm_lexical_substrate/manifest.sop`
- `data/slm_lexical_substrate/term_role_sense_index.json`
- `data/slm_lexical_substrate/closed_class_speed_table.json`
- `data/slm_lexical_substrate/source_provenance.json`

Legacy/debug output:

- `data/lexicon_lookup_cache.json`, generated by `scripts/export_lexicon_cache.py`

Sources:

- `seeds/closed_class_terms.json`
- Open English WordNet XML from `https://en-word.net/`

The SQLite file remains the inspectable authority artifact. The substrate pack is
the SLM's speed-dial runtime artifact: it keeps role candidates, closed-class
structural hints, sense candidates, and source provenance close to the reasoning
loop.

The Rust SLM crate reads the generated substrate index through its `lexicon`
module and turns lexical substrate rows into evidence, not directly into
stabilized glyphs. The substrate is generated from SQLite because this Windows
environment blocks native SQLite build-script execution for direct Rust SQLite
bindings.
